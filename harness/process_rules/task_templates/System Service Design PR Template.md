# System Service Design PR Template

## 1. 模板目标

本模板用于系统服务的 **PR-1 设计 PR**。其目标是冻结边界，而不是直接把代码写完。

适用场景：

- 新增系统服务；
- 对现有系统服务做较大重构；
- 引入新协议、新持久数据格式或新的调度接入方式。

---

## 2. 建议使用方式

将以下内容复制到设计文档或设计 PR 中，并按任务实际情况填写。

---

---

## 4. 使用要求

- 协议设计与持久数据格式设计 **MUST** 在实现前明确；
- 如果某项尚未冻结，**SHOULD** 明确标记为 open question，而不是假装已经确定；
- 若任务包含高风险兼容性变更，**MUST** 在模板中写清升级策略。
