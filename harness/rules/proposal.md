
# 书面开发任务的处理规则

充分阅读任务书
1. 按照我们这份流程，任务书通常属于现有的某个任务处理类型。在这个类型中，其实已经把几个阶段都分好了。
2. 比如一个 Proposal，如果它是关于 UI 的工作，其实内部已经设计好了好几个阶段。
3. 当前需要列出的处理规则及要做的事情如下：
   (a) 确认自己处于一个正确的开发分支上
   (b) 确认现在自己电脑或项目所处的阶段
   (c) 明确当前阶段的任务目标
   确定好自己处于哪一个阶段之后，下一步需要明确：任何书面任务的推进，都必须以当前分支可以发起 Pull Request（PR）作为阶段性结束。

   因此，需要清晰理解当前阶段做到什么程度才可以触发 Pull Request，并提前准备好 PR 相关的文档资料。按照现在的逻辑，一旦该阶段的工作全部完成，分支就可以去做 PR 了。
